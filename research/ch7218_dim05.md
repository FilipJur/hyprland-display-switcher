# Research: Prevent GPU Display Reconfiguration on Monitor Disable

## 1. `amdgpu.dcdebugmask` (DC_DEBUG_MASK) Bits

The `dcdebugmask` module parameter overrides display debug features. All bits are defined in `drivers/gpu/drm/amd/include/amd_shared.h` (enum `DC_DEBUG_MASK`).

### Mainline Bits (up to Linux 6.x)

| Bit Name | Hex Value | Description |
|---|---|---|
| `DC_DISABLE_PIPE_SPLIT` | 0x1 | Disable pipe-splitting across display pipes. |
| `DC_DISABLE_STUTTER` | 0x2 | Disable memory stutter mode. |
| `DC_DISABLE_DSC` | 0x4 | Disable Display Stream Compression (DSC). |
| `DC_DISABLE_CLOCK_GATING` | 0x8 | Disable clock gating optimizations. |
| `DC_DISABLE_PSR` | 0x10 | Disable Panel Self Refresh (PSR) v1 and PSR-SU. |
| `DC_FORCE_SUBVP_MCLK_SWITCH` | 0x20 | Force MCLK switch in SubVP even if vblank switch is possible. |
| `DC_DISABLE_MPO` | 0x40 | Disable multi-plane offloading (MPO). |
| `DC_ENABLE_DPIA_TRACE` | 0x80 | Enable trace logging for DPIA. |
| `DC_ENABLE_DML2` | 0x100 | Force usage of DML2 even if DCN version doesn't default to it. |
| `DC_DISABLE_PSR_SU` | 0x200 | Disable PSR SU. |
| `DC_DISABLE_REPLAY` | 0x400 | Disable Panel Replay. |
| `DC_DISABLE_IPS` | 0x800 | Disable all Idle Power States (IPS), all the time. |
| `DC_DISABLE_IPS_DYNAMIC` | 0x1000 | Disable IPS, except when driver goes into suspend. |
| `DC_DISABLE_IPS2_DYNAMIC` | 0x2000 | Disable IPS2 (IPS1 allowed) if there is an enabled display. |
| `DC_FORCE_IPS_ENABLE` | 0x4000 | Force enable all IPS, all the time. |
| `DC_DISABLE_ACPI_EDID` | 0x8000 | Don't attempt to fetch EDID for eDP from ACPI _DDC method. |
| `DC_DISABLE_HDMI_CEC` | 0x10000 | Disable HDMI-CEC feature. |
| `DC_DISABLE_SUBVP_FAMS` | 0x20000 | Disable DCN Sub-Viewport & Firmware Assisted Memory Clock Switching (FAMS). |
| `DC_DISABLE_CUSTOM_BRIGHTNESS_CURVE` | 0x40000 | Disable support for custom brightness curves. |
| `DC_HDCP_LC_FORCE_FW_ENABLE` | 0x80000 | Use HDCP Locality Check FW path regardless of HW capabilities. |
| `DC_HDCP_LC_ENABLE_SW_FALLBACK` | 0x100000 | Retry HDCP Locality Check using legacy SW path on FW failure. |
| `DC_SKIP_DETECTION_LT` | 0x200000 | Skip detection link training. |

### Out-of-Tree / CachyOS Patch Bits

| Bit Name | Hex Value | Description |
|---|---|---|
| `DC_OVERRIDE_PCON_VRR_ID_CHECK` | 0x400000 | Override PCON (Protocol Converter) VRR ID check. **Not in mainline**; used by CachyOS to work around PCON adapter issues. |

**Command to set:**
```bash
# Add to kernel command line (GRUB, systemd-boot, etc.)
amdgpu.dcdebugmask=0x400000
```

## 2. `amdgpu.dcfeaturemask` (DC_FEATURE_MASK) Bits

Defined in `drivers/gpu/drm/amd/include/amd_shared.h` (enum `DC_FEATURE_MASK`). The default value is `0x2` (bit 1 enabled) in most current kernels.

| Bit Name | Hex Value | Default | Description |
|---|---|---|---|
| `DC_FBC_MASK` | 0x1 | Disabled | Frame Buffer Compression (FBC). |
| `DC_MULTI_MON_PP_MCLK_SWITCH_MASK` | 0x2 | **Enabled** | Allow memory clock (MCLK) switching with multiple monitors in sync. |
| `DC_DISABLE_FRACTIONAL_PWM_MASK` | 0x4 | Disabled | Disable fractional PWM backlight control. |
| `DC_PSR_MASK` | 0x8 | Disabled | Panel Self Refresh (PSR). |
| `DC_EDP_NO_POWER_SEQUENCING` | 0x10 | Disabled | eDP power sequencing bypass. |
| `DC_DISABLE_LTTPR_DP1_4A` | 0x20 | Disabled | Disable DP1.4a Link Training Tuning and Preference Router (LTTPR). |
| `DC_DISABLE_LTTPR_DP2_0` | 0x40 | Disabled | Disable DP2.0 LTTPR. |
| `DC_PSR_ALLOW_SMU_OPT` | 0x80 | Disabled | Allow SMU optimization for PSR. |
| `DC_PSR_ALLOW_MULTI_DISP_OPT` | 0x100 | Disabled | Allow multi-display PSR optimization. |
| `DC_REPLAY_MASK` | 0x200 | Disabled | Panel Replay (for DCN >= 3.1.4). |

**Relevant for preventing reconfiguration:**
- **`DC_MULTI_MON_PP_MCLK_SWITCH_MASK` (0x2)**: When enabled, DC allows MCLK to drop when multiple monitors are connected. **Clearing this bit (setting mask without 0x2) forces MCLK to stay at the highest frequency**, which may prevent some clock reconfiguration events when a monitor is disabled. However, this only applies when multiple monitors are active; once only one monitor remains, the behavior may revert to single-monitor DPM rules.

**Command to set:**
```bash
# To disable multi-monitor MCLK switching (clear bit 1):
amdgpu.dcfeaturemask=0x0
```

## 3. Other Relevant Module Parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `amdgpu.dc` | int | -1 (auto) | Enable/disable Display Core (DC). `0` disables DC entirely (last resort). |
| `amdgpu.dcdebugmask` | uint | 0 | Display debug mask (see Section 1). |
| `amdgpu.dcfeaturemask` | uint | 0x2 | Display feature mask (see Section 2). |
| `amdgpu.forcelongtraining` | uint | 0 | Force long link training on resume. May help with link stability. |
| `amdgpu.ip_block_mask` | uint | 0xffffffff | Disable specific IP blocks. Can be used to disable the display block entirely (heavy-handed). |
| `amdgpu.visualconfirm` | uint | 0 | Visual confirm: `1` = MPO, `5` = PSR. |
| `amdgpu.abmlevel` | uint | 0 | Adaptive Backlight Management level (0-4). |
| `amdgpu.freesync_video` | uint | 0 | Enable FreeSync video mode optimization (experimental). |
| `amdgpu.fakevrrdisplay` | int | 0 | Force all displays to be detected as VRR capable (testing only). |

## 4. Debugfs / Sysfs Interfaces

### `force_timing_sync`
- **Path**: `/sys/kernel/debug/dri/0/amdgpu_dm_force_timing_sync` (or similar, depending on DRM minor)
- **Type**: debugfs file (write `1` to enable)
- **Effect**: Forces all connected displays to synchronize timing. This is **not** a module parameter; it must be written at runtime.
- **Command**:
  ```bash
  echo 1 | sudo tee /sys/kernel/debug/dri/0/amdgpu_dm_force_timing_sync
  ```

### `disable_hpd_irq`
- **Status**: `bool` inside `struct amdgpu_display_manager`.
- **Exposure**: **NOT EXPOSED** as a module parameter or debugfs/sysfs in mainline.
- **Effect**: Setting this to `true` disables all HPD (Hot Plug Detect) and HPD RX interrupt handling, preventing the driver from reacting to monitor disconnect/connect events.
- **Workaround**: A kernel patch is required to expose this, or HPD must be suppressed at the hardware level (e.g., by keeping the HPD line high with a pull-up resistor or using a hardware device like DPHPDMA).

### `power_dpm_force_performance_level`
- **Path**: `/sys/class/drm/card0/device/power_dpm_force_performance_level`
- **Effect**: Forces GPU performance level to `high`, `low`, `auto`, etc. This controls GPU core/memory clocks, not display pipeline clocks.
- **Relevance**: Indirect; may reduce overall DPM-related state changes.

## 5. Parameters to Prevent Display Reconfiguration on Monitor Disable

### Direct Parameters
**There is NO single module parameter in mainline amdgpu that directly prevents display pipeline reconfiguration when a monitor is disabled.** The driver's DC layer is designed to re-optimize the pipeline on any hotplug/disable event.

### Indirect Approaches

#### A. Disable HPD Interrupts (Most Effective)
- **Method**: Set `adev->dm.disable_hpd_irq = true`.
- **Current Status**: Not exposed; requires a kernel patch or runtime modification.
- **Hardware Alternative**: Use a DisplayPort HotPlug maintainer (DPHPDMA) or a 10k-20k pull-up resistor between DP_PWR (pin 20) and HPD (pin 18) on the GPU side to keep HPD high, making the GPU think the monitor is always connected. This prevents the driver from ever seeing a disconnect event.

#### B. Skip Detection Link Training
- **Bit**: `DC_SKIP_DETECTION_LT` (0x200000)
- **Command**:
  ```bash
  amdgpu.dcdebugmask=0x200000
  ```
- **Effect**: Skips detection link training. May reduce some reconfiguration overhead during hotplug events, but does not stop the driver from reconfiguring the display core.

#### C. Disable Clock Gating
- **Bit**: `DC_DISABLE_CLOCK_GATING` (0x8)
- **Command**:
  ```bash
  amdgpu.dcdebugmask=0x8
  ```
- **Effect**: Disables clock gating optimizations. This may prevent some clock domain reconfiguration when the pipeline changes, but does not prevent the pipeline reconfiguration itself.

#### D. Disable Idle Power States
- **Bits**: `DC_DISABLE_IPS` (0x800), `DC_DISABLE_IPS_DYNAMIC` (0x1000), `DC_DISABLE_SUBVP_FAMS` (0x20000)
- **Command**:
  ```bash
  amdgpu.dcdebugmask=0x20800
  ```
- **Effect**: Keeps display power states from entering low-power modes, which may reduce the number of state transitions when a monitor is disabled.

#### E. Force MCLK High (Multi-Monitor)
- **Bit**: Clear `DC_MULTI_MON_PP_MCLK_SWITCH_MASK` (0x2) in `dcfeaturemask`.
- **Command**:
  ```bash
  amdgpu.dcfeaturemask=0x0
  ```
- **Effect**: Forces MCLK to stay at highest frequency when multiple monitors are connected. May prevent MCLK reconfiguration, but the display pipeline may still be re-optimized.

#### F. Disable DC Entirely
- **Parameter**: `amdgpu.dc=0`
- **Effect**: Disables the Display Core. You will lose all display output. **Last resort only.**

#### G. Disable Display IP Block
- **Parameter**: `amdgpu.ip_block_mask`
- **Effect**: Can be used to disable the display IP block entirely. Also results in loss of display output. **Not recommended.**

## 6. `force_clock_mode` / `forced_clocks`

- **Location**: `struct dc_debug_options` (internal to DC).
- **Exposure**: **NOT FOUND** as a module parameter, sysfs, or debugfs entry in mainline.
- **Usage**: The field is used internally by clock manager code (e.g., `dcn201_clk_mgr.c`) to force a specific clock mode for debugging. There is no runtime interface to set it without modifying the kernel source.

## 7. Summary of Commands

### Preventing Reconfiguration (Best Effort)
```bash
# /etc/modprobe.d/amdgpu-prevent-reconfig.conf
options amdgpu dcdebugmask=0x200008  # DC_SKIP_DETECTION_LT | DC_DISABLE_CLOCK_GATING
options amdgpu dcfeaturemask=0x0       # Disable multi-monitor MCLK switching
options amdgpu forcelongtraining=1     # Force long training on resume
```

### Disabling HPD (Requires Patch or Hardware)
- **Software**: Patch `drivers/gpu/drm/amd/display/amdgpu_dm/amdgpu_dm.c` to expose `disable_hpd_irq` via a module parameter or debugfs.
- **Hardware**: Insert a 10k-20k resistor between DP_PWR and HPD on the GPU-side connector to keep HPD high.

### CachyOS-Specific PCON Fix
```bash
# CachyOS kernel includes DC_OVERRIDE_PCON_VRR_ID_CHECK (0x400000)
options amdgpu dcdebugmask=0x400000
```

---

## 8. NOT FOUND

- **No module parameter** named `amdgpu.disable_display_reconfig`, `amdgpu.disable_hotplug`, or similar.
- **No sysfs/debugfs path** to directly set `disable_hpd_irq` or `force_clock_mode` in mainline.
- **No bit** in `DC_DEBUG_MASK` or `DC_FEATURE_MASK` that explicitly says "disable display pipeline re-optimization on monitor disable."
- **No official documentation** describing a knob to "pin" a display configuration across hotplug events.
